## 二、质控指标与系统计算符对应映射

### 2.1 已完全覆盖（可直接配置）

| 质控指标 | 实现方法关键词 | 对应计算符 | 配置要点 |
|----------|--------------|-----------|---------|
| 缺现病史 | 为空校验 | `isBlank` | 直接校验字段空值 |
| 缺专科情况描述 | 为空校验 | `isBlank` | 直接校验字段空值 |
| 输血记录缺输血效果及后续记录 | 为空校验 | `isBlank` | 直接校验字段空值 |
| 无主持人小结 | 小于10个字符 | `lengthCheck` | 比较符`<`，阈值10 |
| 无病情摘要具体内容 | 小于30个字符 | `lengthCheck` | 比较符`<`，阈值30 |
| 主诉与现病史症状不一致 | 交集个数不一致 | `arrayIntersect` | 阈值0，取两个字段的症状集合比对 |
| 现病史与主诉症状不一致 | 交集为空 | `arrayIntersect` | 阈值0 |
| 门诊主诉与住院主诉症状不一致 | 交集为空 | `arrayIntersect` | 阈值0 |
| 二次以上病程记录完全相同 | 相似度大于0.99 | `similarity` | 阈值0.99，对比字段选"其他病程记录" |
| 首次病程涉嫌拷贝 | 相似度大于0.995 | `similarity` | 阈值0.995 |
| 交班与接班记录雷同 | 相似度比对 | `similarity` | 阈值0.95 |
| 转出和转入记录雷同 | 相似度比对 | `similarity` | 阈值0.95 |
| 疑难病例与术前讨论雷同 | 相似度比对 | `similarity` | 阈值0.90 |
| 辅助检查抄写有缺陷 | 相似度比对 | `similarity` | 阈值0.95 |
| 体格检查脉搏与心率矛盾 | 数值比较 | `dataCheck` | 根据诊断判断是否房颤，选择比较符 |
| 门诊主诉与住院主诉持续时间不一致 | 正则提取后比较 | `regex_match` + `timeCheck` | 先用正则提取时间，再用timeCheck比对 |
| 主诉无持续时间描述 | 正则匹配 | `regex_match` | 正则：`.*(时\|月\|周\|天\|年\|日).*` |
| 起病时间描述不准确 | 多正则匹配 | `regex_match` / `multiRegexMatch` | 5条时间格式正则，任一匹配即通过 |
| 现病史缺症状时间描述 | 正则匹配 | `regex_match` | 匹配时间量词+时间单位 |
| 现病史缺症状性质描述 | 关键词正则 | `regex_match` | 匹配"时\|程度\|性质\|部位\|颜色\|诱因\|表现\|状态" |
| 现病史缺症状程度描述 | 关键词正则 | `regex_match` | 匹配"程度\|性质\|范围\|发作频率\|轻度\|重度\|中度" |
| 现病史缺一般情况描述 | 关键词正则 | `regex_match` | 匹配"二便\|精神\|饮食\|睡眠\|大便\|小便"等 |
| 现病史缺发病后诊治情况 | 关键词正则 | `regex_match` | 匹配"进一步\|当地医院\|就诊\|未系统诊治"等 |
| 现病史缺疾病发展变化描述 | 关键词正则 | `regex_match` | 匹配"加重\|缓解\|出现\|消失\|新增"等 |
| 诊断依据有缺陷 | 关键词正则 | `regex_match` | 排除"排除\|可能\|不符合\|明确"等否定词 |
| 诊疗计划有缺陷 | 关键词正则 | `regex_match` | 匹配"完善\|治疗\|对症\|进一步" |
| 首次查房未反映上级意见 | 关键词正则 | `regex_match` | 匹配"追问病史\|无补充\|查体\|补充诊断"等 |
| 辅助检查缺时间描述 | 日期正则 | `regex_match` | 匹配日期格式 |
| 辅助检查外院未注明医院 | 正则匹配 | `regex_match` | 匹配"医院\|实验室\|中心\|院\|所\|科" |
| 危急值记录无报告时间 | 正则+时间 | `regex_match` + `timeCheck` | 先正则定位时间字段，再校验 |
| 体格检查遇漏标志性阳性体征 | 正则匹配 | `regex_match` | 匹配"右下腹\|麦氏点\|压痛\|墨菲氏征"等 |
| 缺有鉴别诊断意义的阴性体征 | 集合长度+否定检测 | `arrayLength` + `negationCheck` | 提取阴性体征集合，长度<5且含否定词则质控 |
| 缺有鉴别诊断意义的阴性症状 | 集合长度+否定检测 | `arrayLength` + `negationCheck` | 同上 |
| 主诉无症状描述 | 医学实体提取 | `medicalNer` | 实体类型选"症状"，检测是否为空 |
| 现病史缺伴随症状 | 医学实体提取 | `medicalNer` | 实体类型选"症状"，检测是否为空 |
| 体格检查遗漏主要阳性体征 | 医学实体提取 | `medicalNer` | 实体类型选"体征"，结合科室过滤 |
| 现病史发病诱因描述不清 | 实体+正则 | `medicalNer` + `regex_match` | 先提取症状实体，再正则匹配治疗关键词 |
| 主诉用诊断代替但现病史有症状 | 实体提取+矛盾检测 | `medicalNer` + `contradictionCheck` | 主诉提取诊断实体，现病史提取症状实体，对比 |
| 现病史里的主要症状和主诉不一致 | 实体提取+集合交集 | `medicalNer` + `arrayIntersect` | 分别提取症状集合，计算交集 |
| 患者病情描述前后矛盾 | 矛盾检测 | `contradictionCheck` | 配置否定词库和关键字库 |
| 主诉与伴随症状矛盾 | 矛盾检测 | `contradictionCheck` | 如"昏迷"与"神志清醒"矛盾 |
| 过敏史前后矛盾 | 否定检测+矛盾 | `negationCheck` + `contradictionCheck` | 先剔除否定描述，再检测阳性实体 |
| 个人史与既往诊断矛盾 | 矛盾检测 | `contradictionCheck` | 如诊断"酒精中毒"但个人史写"否认酗酒" |
| 药物过敏史与既往登记不一致 | 存在性冲突 | `existenceConflict` | 对比两份病历的过敏药品集合 |
| 辅助检查与诊断相关项目未描述 | 字典匹配 | `dictMatch` | 诊断关联字典，检查辅助检查是否包含 |
| 体格检查与主诉/诊断相关项目未描述 | 字典匹配 | `dictMatch` | 诊断关联字典，检查体格检查是否包含 |
| 使用恶性肿瘤化疗药物未记录 | 字典匹配 | `dictMatch` | 恶性肿瘤诊断字典+化疗药物字典 |
| 使用恶性肿瘤放疗未记录 | 字典匹配 | `dictMatch` | 恶性肿瘤诊断字典+放疗关键字字典 |

### 2.2 部分覆盖（需要组合或增强）

| 质控指标 | 当前计算符局限 | 需要的增强 |
|----------|---------------|-----------|
| 入院时间与病史采集时间差>2小时 | `timeCheck` 只支持单字段时间差 | 需要支持**两个字段交叉比对**（如入院时间 vs 病史采集时间） |
| 长期医嘱开始时间<病史采集时间 | `timeCheck` 不支持医嘱数据 | 需要接入**医嘱数据**作为比对源 |
| 缺症状部位描述 | `medicalNer` 提取症状但无部位属性 | 需要 NLP 实体提取支持**症状-部位**关系抽取 |
| 现病史缺发病后诊治情况 | `medicalNer` 可提取药品但无医嘱联动 | 需要**与医嘱系统联动**，校验是否真有医嘱 |
| 体格检查顺序颠倒 | 无对应计算符 | 需要新增**顺序校验**（如按头→颈→胸→腹顺序） |
| 初步诊断主次排序有误 | 无对应计算符 | 需要新增**诊断优先级排序校验**（如主要诊断应排第一） |
| 存在漏诊 | 无对应计算符 | 需要**诊断与检查/体征的完整性关联规则** |
| 未预防和及时诊断 | 无对应计算符 | 需要**时间轴+诊断触发规则** |
| 参与人员不符合管理要求 | 无对应计算符 | 需要**人员角色+数量规则**（如术前讨论需≥3人） |
| 表格病历体格检查记录有漏项 | `isBlank` 只能校验整个字段 | 需要**子字段空值校验**（表格中某几列为空） |

### 2.3 暂无法覆盖（需新增功能）

| 质控指标 | 所需新功能 | 优先级 |
|----------|-----------|--------|
| 长期医嘱开始时间<病史采集时间 | **医嘱数据接入** + 时间交叉比对 | 高 |
| 入院时间与病史采集时间差 | **双字段时间差计算**（支持绝对值/方向） | 高 |
| 体格检查顺序颠倒 | **文本顺序校验**（按预设模板顺序匹配） | 中 |
| 初步诊断主次排序有误 | **列表顺序校验**（诊断列表第一项必须是主要诊断） | 中 |
| 存在漏诊/主要诊断不充分 | **诊断完整性推理**（诊断→检查→体征闭环校验） | 低 |
| 参与人员不符合管理要求 | **人员角色计数**（提取人名+职称，计数校验） | 低 |
| 表格病历子字段漏项 | **结构化子字段校验**（JSON/表格字段级空值检测） | 中 |
| 未预防和及时诊断 | **时间轴规则引擎**（事件A发生后N小时内必须出现事件B） | 低 |

## 四、提升建议

### 4.1 新增计算符（按优先级排序）

#### P0-高优先级：双字段时间差计算符

**场景**：入院时间 vs 病史采集时间差、长期医嘱时间 vs 病史采集时间、脉搏 vs 心率值比对。

**建议新增**：`timeDiff` / `fieldCompare`
```
输入：
  - 字段A（如入院时间）
  - 字段B（如病史采集时间）
  - 比较符（> / < / ==）
  - 阈值（如2，单位小时）
输出：|字段A - 字段B| 与阈值比较
```

#### P1-中优先级：文本顺序校验

**场景**：体格检查应按"一般情况→皮肤→淋巴结→头部→颈部→胸部→腹部→四肢→神经系统"顺序记录。

**建议新增**：`sequenceCheck`
```
输入：
  - 字段（如体格检查全文）
  - 期望顺序关键词列表（如["一般情况","皮肤","淋巴结","头部","颈部","胸部","腹部","四肢","神经系统"]）
输出：是否按顺序出现，缺失/错位项列表
```

#### P2-中优先级：列表顺序校验

**场景**：初步诊断列表中，主要诊断必须是第一项。

**建议新增**：`listOrderCheck`
```
输入：
  - 字段（如诊断列表JSON）
  - 主字段属性名（如"isPrimary"）
输出：主项索引是否为0
```

#### P3-低优先级：人员角色计数

**场景**：术前讨论参与人员需≥3人，且含主任医师。

**建议新增**：`roleCountCheck`
```
输入：
  - 字段（如讨论记录全文）
  - 角色正则列表（如{"主任医师": ".*主任医师.*", "副主任医师": ".*副主任医师.*"}）
  - 最少人数要求
输出：各角色匹配人数是否达标
```

### 4.2 增强现有计算符

#### 增强1：`timeCheck` 支持双字段比对

当前 `timeCheck` 只能比较一个字段与基准时间。建议增强为支持两个字段互相比较：
```jsx
<Form.Item name="value" label="基准字段A" />
<Form.Item name="extraValue1" label="对比字段B" />
<Form.Item name="extraValue2" label="最小差值（小时）" />
<Form.Item name="extraValue3" label="最大差值（小时）" />
```

#### 增强2：`medicalNer` 支持部位关联

当前 `medicalNer` 提取独立实体。建议增加 `entityRelation` 模式，返回实体关系：
```json
{
  "entities": [
    {"type": "symptoms", "text": "腹痛", "position": [10, 12]},
    {"type": "signs", "text": "右下腹", "position": [15, 18]}
  ],
  "relations": [
    {"source": "腹痛", "target": "右下腹", "type": "部位"}
  ]
}
```

#### 增强3：`contradictionCheck` 支持跨字段

当前 `contradictionCheck` 只能检测单字段内的矛盾。建议支持配置两个对比字段：
```jsx
<Form.Item name="value" label="字段A否定词库" />
<Form.Item name="extraValue1" label="字段A关键字库" />
<Form.Item name="extraValue2" label="字段B名称" />  // 新增：跨字段比对
```

### 4.3 数据接入建议

| 数据源 | 对接指标 | 实现方式 |
|--------|---------|---------|
| 医嘱数据 | 长期医嘱时间校验、用药记录校验 | 通过 `valueSource=ADAPTER` 接入医嘱查询接口 |
| 门诊病历 | 门诊主诉vs住院主诉一致性 | 通过 `valueSource=SQL` 查询门诊病历库 |
| 历史病历 | 过敏史一致性、病程拷贝检测 | 通过 `valueSource=SQL` 查询历史病历库 |
| 人员信息 | 参与人员职称校验 | 通过 `valueSource=ADAPTER` 接入人事系统 |

### 4.4 配置优化建议

1. **规则模板化**：将病历内涵质控的 86 条指标固化为规则模板，一键导入生成规则（已有 `generate_qc_rules_skill.py` 基础，可扩展）
2. **条件模型批量同步**：当前 `ConditionSyncService` 只同步数据元到条件模型，建议增加**病历字段→条件模型**的批量同步
3. **字典扩充**：
   - 增加"症状-部位"关联字典
   - 增加"诊断-必查项目"关联字典（如"骨折"→"X片/CT/MR"）
   - 增加"科室-排除规则"字典（如放射科不校验体格检查）
4. **结果分级**：当前结果只有类型和优先级，建议增加**质控等级**（如"一票否决"/"一般缺陷"/"提示"）

## 五、快速落地清单

### 本周可配置（无需开发）

使用现有计算符直接配置的指标：

| 指标 | 计算符 | 配置参数 |
|------|--------|---------|
| 缺现病史 | `isBlank` | 字段=现病史 |
| 无主持人小结 | `lengthCheck` | 比较符`<`，阈值10，字段=主持人小结 |
| 主诉与现病史症状不一致 | `arrayIntersect` | 对比字段=主诉症状集合，阈值=0 |
| 病程记录完全相同 | `similarity` | 阈值0.99，对比字段=其他病程记录 |
| 体格检查脉搏与心率矛盾 | `dataCheck` | 操作符`!=`，阈值=0，字段=脉搏-心率 |
| 主诉无持续时间 | `regex_match` | 正则=`.*(时\|月\|周\|天\|年\|日).*` |
| 缺一般情况描述 | `regex_match` | 正则=`.*(二便\|精神\|饮食\|睡眠\|大便\|小便).*` |
| 缺伴随症状 | `medicalNer` | 实体类型=症状，字段=现病史 |
| 过敏史前后矛盾 | `contradictionCheck` | 否定词=否认药物过敏/否认食物过敏，关键字=过敏 |
| 药物过敏史不一致 | `existenceConflict` | 冲突关键词=上一病历过敏药品，对比字段=既往史 |

### 需开发后配置（1-2周）
